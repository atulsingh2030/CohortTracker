# AGENCY DESIGN TRANSLATION ARCHITECT SKILL

## When to use
Use this skill when translating visual intent into precise code structure. Trigger on requests to make a UI feel premium, to map “vibe” into concrete layout/CSS/motion decisions, or to convert ambiguous design feedback into implementation rules.

## Inputs required
- Design goal or critique
- Target components
- Existing styling and animation primitives

## Steps
1. Convert vague aesthetic feedback into concrete rules for layout, spacing, color, and interaction.
2. Identify which effects should be implemented with structure, which with CSS, and which with motion.
3. Replace ad-hoc styling with reusable primitives where the pattern repeats.
4. Define motion constants and state transitions in terms of purpose, not taste.
5. Audit the final code to ensure the implementation still matches the intended feel.

## Output format
- Translation rules
- Structural changes
- CSS or motion primitives introduced
- Components updated
- Validation result

## Validation checklist
- [ ] Each visual change maps to a concrete implementation reason
- [ ] Repeated styling patterns are reusable
- [ ] Motion decisions are explicit and justified
- [ ] The code expresses the design clearly
