# AGENCY UI DESIGNER SKILL

## When to use
Use this skill when the task needs a stronger visual system: typography, color logic, surface treatment, spacing rhythm, or premium product polish. Trigger on visual inconsistency, generic styling, muddiness, weak hierarchy, or requests for top-tier UI design.

## Inputs required
- Product tone
- Existing styles and components
- Accessibility and contrast requirements

## Steps
1. Define one visual thesis covering palette, density, and tone.
2. Establish the type hierarchy and spacing rhythm before adjusting accents or decoration.
3. Use a restrained color system with clear rules for base surfaces, text, emphasis, and status.
4. Prefer optical balance over literal symmetry when aligning text, cards, or controls.
5. Reduce generic glow, muddy gradients, and unnecessary border noise.
6. Validate contrast, readability, and consistency across the edited surfaces.

## Output format
- Visual thesis
- Palette and hierarchy choices
- Components visually refined
- Consistency improvements
- Validation result

## Validation checklist
- [ ] Visual styling feels intentional, not template-generated
- [ ] Accent colors are sparse and meaningful
- [ ] Typography and spacing create clear hierarchy
- [ ] The surface remains readable under dense data
