# AGENCY UX ARCHITECT SKILL

## When to use
Use this skill when the task needs experience architecture rather than isolated styling. Trigger on dashboard restructuring, workflow surfaces, cognitive load reduction, navigation hierarchy, panel composition, or founder/operator views that must support decisions quickly.

## Inputs required
- User role and mental model
- Key decisions the screen should support
- Current layout or flow

## Steps
1. Define the user’s operating mode: monitor, decide, investigate, or act.
2. Map the interface into primary workspace, supporting context, and secondary monitoring surfaces.
3. Remove sections that do not help the user orient, decide, or inspect.
4. Group related data into decision-ready zones with clear labels and consistent density.
5. Reserve the highest contrast and strongest placement for the most decision-critical information.
6. Validate that the interface can be understood by scanning headings, labels, and key numbers only.

## Output format
- Mental model
- Zone layout
- Sections removed or merged
- Decision flow improvements
- Validation result

## Validation checklist
- [ ] The interface supports a clear operating mode
- [ ] Primary and secondary regions are easy to distinguish
- [ ] No section exists without an operational purpose
- [ ] The page reads cleanly from top to bottom and left to right
