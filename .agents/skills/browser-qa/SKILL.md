# BROWSER QA SKILL

## When to use
Use this skill when the task is to visually inspect the app in a browser, capture screenshots, or verify layout behavior.
Keywords: qa, screenshot, responsive, layout, browser.

## Inputs required
- URL or local port
- Key user flows or views to inspect

## Steps
1. Open the target page.
2. Check loading state, errors, and key UI sections.
3. Check desktop and mobile layout if relevant.
4. Capture evidence for anything claimed.

## Output format
- URLs checked
- Screens verified
- Issues found
- Evidence captured

## Validation checklist
- [ ] Claims are backed by direct browser evidence
- [ ] Layout issues are described concretely
- [ ] No browser claim is made without verification
