# CODE REVIEWER SKILL

## When to use
Use this skill when the task is to review a diff, audit a change, or call out risks before merge.
Keywords: review, audit, quality, check this diff.

## Inputs required
- Files or diff to review
- Expected behavior

## Steps
1. Read the changed files.
2. Identify correctness, regression, and maintainability risks.
3. Prioritize findings by severity.
4. Call out missing verification where relevant.

## Output format
- Findings first
- Open questions or assumptions
- Short summary only after findings

## Validation checklist
- [ ] Focus stayed on bugs and risks
- [ ] Findings are specific
- [ ] No invented issues
