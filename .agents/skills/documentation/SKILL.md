# DOCUMENTATION SKILL

## When to use
Use this skill when the task is to write, rewrite, organize, or simplify repo documentation.
Keywords: docs, README, explain, guide, write down.

## Inputs required
- Target audience
- Relevant product or code context
- File paths to update

## Steps
1. Verify the product or code behavior from the repo.
2. Identify the smallest set of docs that should change.
3. Write for clarity and fast comprehension.
4. Remove stale claims and legacy references.
5. Re-read for consistency with the current repo state.

## Output format
- Audience
- Files changed
- Claims verified
- Remaining gaps

## Validation checklist
- [ ] Claims match the repo
- [ ] Legacy references were removed or called out
- [ ] Setup steps are readable
- [ ] No speculative claims were added
