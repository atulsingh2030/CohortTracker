# IMPLEMENTATION SKILL

## When to use
Use this skill when the task requires writing, building, or changing something concrete in code or configuration.
Keywords: build, write, create, implement, code, add feature, make.

## Inputs required
- Requirements
- Tech stack or runtime
- Relevant file paths or modules

## Steps
1. Confirm the requirements and assumptions.
2. Identify the files or modules that need changes.
3. Choose the smallest implementation that satisfies the request.
4. Apply the change.
5. Check edge cases and adjacent behavior.
6. Run an appropriate verification step.
7. Summarize what changed and why.

## Output format
- Approach summary
- Files changed
- Edge cases addressed
- Verification result
- One-line change summary

## Validation checklist
- [ ] Requirements were satisfied
- [ ] No dead code was introduced
- [ ] Existing behavior was considered
- [ ] The result is readable and maintainable
