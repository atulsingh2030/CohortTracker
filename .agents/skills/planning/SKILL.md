# PLANNING SKILL

## When to use
Use this skill when the task is primarily about strategy, architecture, sequencing, or thinking before execution.
Keywords: plan, roadmap, design, architect, strategy, approach, figure out.

## Inputs required
- Objective
- Constraints
- Known unknowns or blockers

## Steps
1. Restate the objective in one sentence.
2. List the known constraints and open questions.
3. Break the objective into discrete tasks.
4. Order the tasks by dependency.
5. Identify risks for each major task.
6. Produce the structured plan.
7. Validate that the plan covers the objective without orphaned steps.

## Output format
- Objective
- Constraints
- Ordered task list
- Dependency notes
- Risk log with mitigations

## Validation checklist
- [ ] Objective was restated correctly
- [ ] Each task has a dependency anchor
- [ ] Each major risk has a mitigation
- [ ] The plan is reviewable before execution starts
