# SENIOR FRONTEND UI UX ENGINEER SKILL

## When to use
Use this skill when building or refining production frontend surfaces that must feel polished, deliberate, and interaction-safe. Trigger on premium dashboards, complex responsive layouts, component refinement, UI cleanup, animation tuning, or requests for top-tier frontend quality.

## Inputs required
- Screen or feature goal
- Existing component files
- Data shape and interaction constraints

## Steps
1. Identify the primary working surface and remove any layout or chrome that distracts from it.
2. Tighten hierarchy through spacing, type scale, contrast, and information grouping before adding decoration.
3. Make responsive behavior explicit for desktop, tablet, and mobile rather than relying on accidental stacking.
4. Use motion only for orientation, emphasis, or state change. Keep transitions fast and justified.
5. Audit affordances: active state, hover state, loading state, empty state, and failure state.
6. Validate readability, density, and scan-ability in the final layout.

## Output format
- Visual direction
- Components changed
- Interaction states improved
- Responsive concerns addressed
- Verification result

## Validation checklist
- [ ] The interface feels like a product surface, not a marketing page
- [ ] Information hierarchy is obvious within 3 seconds of scanning
- [ ] Mobile layout is intentional and does not overflow
- [ ] Motion supports understanding instead of adding noise
- [ ] Empty and loading states are still coherent
