# IMPLEMENTATION SKILL

## When to use
When the task requires writing, building, or creating something concrete.
Keywords: build, write, create, implement, code, add feature, make.

## Inputs required
- Requirements (what to build)
- Tech stack / language
- Any existing code context (file paths, function signatures)

## Steps
1. Confirm requirements are unambiguous — if not, surface questions before proceeding
2. Identify which files / modules will be touched
3. Choose implementation approach (and briefly note why)
4. Write the solution in the smallest working increment
5. Check for edge cases not covered by the requirements
6. Validate the output compiles / runs / passes basic correctness check
7. Write a one-line summary of what was changed and why

## Output format
- Approach summary (2–3 lines)
- Code changes (with file paths)
- Edge cases addressed
- One-line change summary

## Validation checklist
- [ ] Requirements satisfied (not just approximately)
- [ ] No dead code introduced
- [ ] Existing functionality not broken
- [ ] Code is readable without needing a comment on every line
